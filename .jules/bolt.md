## 2026-05-18 - [Optimizing SSE Stream Parsers in Dart]
**Learning:** Manual string concatenation (`StringBuffer` and `indexOf('\n')`) for parsing Server-Sent Events (SSE) from `http.StreamedResponse` is inefficient (creates many intermediate string objects and causes memory allocations/UI jank). Furthermore, manually decoding chunks with `utf8.decode(bytes)` incorrectly handles multi-byte UTF-8 characters (like 'ñ', 'ó') that happen to get split across TCP chunk boundaries, rendering as malformed bytes ().
**Action:** Use Dart's native stream transformers (`byteStream.transform(const Utf8Decoder(allowMalformed: true)).transform(const LineSplitter())`) instead. It is significantly faster, reduces garbage collection overhead, and correctly buffers multi-byte characters split across network chunks.

## 2026-05-18 - [Isolating Stream State to Prevent Heavy List Re-renders]
**Learning:** In Flutter, updating a streaming response using `setState` inside an `await for` loop that modifies a string buffer causes the entire screen (including complex widgets like `ListView.builder` and `MarkdownBody` instances within previous messages) to rebuild for *every single incoming network chunk*. This leads to O(N * M) widget rebuilds and severe UI jank, especially as the message history grows.
**Action:** Use a `ValueNotifier` to hold the fast-updating stream buffer state, and wrap *only* the single widget that needs to display it (e.g., the streaming `ChatBubble`) in a `ValueListenableBuilder`. This confines the rebuild exclusively to the UI element displaying the incoming text, eliminating N*M re-renders of the entire view tree.
